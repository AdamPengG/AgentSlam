# agentslam-onemaplike-roadmap

Use when planning the transition from the explicit GT semantic map to a more
online, OneMap-like baseline.

## Trigger

- roadmap updates
- architecture comparison
- decomposition of Phase 1A to 1B work

## Workflow

1. treat the current explicit semantic map as the base, not throwaway code
2. propose incremental upgrades in queryability, persistence, and online use
3. write plans and decision notes before rewriting runtime code

## Outputs

- roadmap text
- acceptance deltas
- dependencies that remain deferred
