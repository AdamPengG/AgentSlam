# Handoff Playbook

## What Every Milestone Should Leave Behind

- updated docs for plans, interfaces, dataflow, and evaluation
- a short milestone report in `reports/`
- runnable commands for the smallest useful demo
- a clear list of remaining gaps and risks

## Reviewer-First Packaging

- lead with what changed
- show how to run the minimal demo
- point to the first two or three reports worth reading
- separate blocking gaps from nice-to-have follow-ups

## Suggested Milestone Template

1. scope completed
2. commands run
3. evidence produced
4. open gaps
5. next recommended phase step

## Phase-to-Phase Handoff Rule

- Phase 1 should hand off a working semantic baseline before Phase 2 asks for richer room graphs or navigation behavior
- later work should build on the verified fixture and export path instead of replacing it without evidence
